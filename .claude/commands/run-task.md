# 検証タスクの実行

指定されたタスク番号の検証タスクを実行する。

## 使い方

`/run-task <task_number>` — タスク番号を指定して実行（例: `/run-task 00`, `/run-task 02`）

引数なしの場合は `docs/tasks/progress.json` を確認し、次に実行すべきタスクを自動判定する。

## 実行手順

1. `docs/tasks/progress.json` を確認し、依存タスクが完了しているかチェック
2. 対応する `docs/tasks/<番号>-*.md` を読み、手順を把握
3. 必要なスクリプトファイルが `libs/*/` に存在するか確認。なければ md の内容に基づいて作成
4. スクリプトを実行（`cd libs/<name> && uv run python <script>.py`）
5. 実行結果を確認し、`progress.json` を更新
6. エラーがあれば原因を調査し、可能なら修正して再実行

## タスク番号と内容

- `00` — 環境構築（uv sync, .gitignore, ディレクトリ作成）
- `01` — データセット準備（SDV demo データのダウンロード）
- `02` — Phase 1: 単一表実験（4ライブラリ + 評価3種）
- `03` — Phase 2: 複数表実験（SDV + MOSTLY AI + 評価）
- `04` — Phase 3: 時系列実験（4ライブラリ + 評価）
- `05` — 統合評価・結果JSON生成

## 重要ルール

- 各ライブラリのスクリプトは対応する `libs/<name>/` ディレクトリで `uv run` を使って実行
- sudo は絶対に使わない
- エラーが発生した個別ライブラリはスキップして次に進む
- インタラクティブな入力を求めない（API キー未設定なら自動スキップ）
- 実行完了後、結果のサマリを表示する
